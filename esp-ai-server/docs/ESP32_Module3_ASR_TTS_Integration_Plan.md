# ESP32模块3规划：ASR/TTS服务集成模块

## 📋 模块概述

**模块3目标**: 创建ASR/TTS服务集成模块，将语音识别和文本转语音服务无缝集成到ESP32新架构中，实现完整的语音交互流程。

## 🎯 模块目标

### 核心目标
1. **ASR服务集成**: 将现有ASR服务集成到新的音频处理架构中
2. **TTS服务集成**: 将现有TTS服务集成到新的音频发送架构中
3. **意图处理器**: 参考`core/handle/intentHandler.py`创建模块化的意图处理系统
4. **服务优化**: 解决ASR连接重复建立/关闭、空音频帧等问题
5. **完整流程**: 实现从音频接收到TTS回复的完整语音交互流程

### 技术目标
- 模块化设计，易于测试和维护
- 异步处理，支持高并发
- 错误恢复和资源管理
- 与模块1和模块2的完美集成
- 性能优化和资源节约

## 🏗️ 架构设计

### 设计原则
1. **参考开源模式**: 借鉴`core/handle`目录的优秀设计模式
2. **模块化分离**: 每个组件职责单一，接口清晰
3. **异步优先**: 所有服务调用都采用异步模式
4. **错误处理**: 完善的异常处理和降级机制
5. **资源管理**: 连接池、缓存和自动清理

### 核心组件

## 🔄 处理流程

### 完整语音交互流程
```
1. 音频接收 (模块2)
   ↓
2. VAD检测 (模块2)
   ↓
3. ASR识别 (模块3) → ASR服务集成器
   ↓
4. 意图处理 (模块3) → 意图处理器
   ↓
5. LLM生成 (模块3) → 意图处理器
   ↓
6. TTS合成 (模块3) → TTS服务集成器
   ↓
7. 音频发送 (模块2)
```

### ASR处理流程
```
音频帧 → 格式验证 → 静音检测 → ASR调用 → 结果验证 → 意图处理
```

### TTS处理流程
```
文本 → 预处理 → TTS调用 → 格式转换 → 分帧 → 协议编码 → 发送
```

## 🔧 需要解决的关键问题

### 1. ASR服务问题
**问题**: ASR连接重复建立/关闭，空音频帧导致无限调用
**解决方案**:
- 实现ASR连接池和复用机制
- 添加音频数据有效性检查
- 优化静音检测和过滤
- 实现智能重试和超时控制

### 2. TTS服务问题
**问题**: 音频格式不匹配，字节序问题，帧长配置错误
**解决方案**:
- 统一音频格式转换接口
- 实现大端字节序支持
- 动态帧长配置和适配
- 音频质量验证和优化

### 3. 意图处理问题
**问题**: 缺少模块化的意图处理框架
**解决方案**:
- 参考intentHandler.py创建模块化处理器
- 支持多种对话模式和function_call
- 集成上下文管理和对话历史
- 实现智能回复生成和优化

### 4. 服务集成问题
**问题**: 各服务间缺少统一的协调机制
**解决方案**:
- 创建语音交互协调器
- 实现状态管理和流程控制
- 支持并发处理和资源调度
- 提供完整的错误恢复机制
